    ### Step B — Infer the document type (D4-04 decision tree)

    Run the source through this decision tree. The first rule that fires wins. Default-on-ambiguous lands `handbook`.

    ```
    1. Source contains 2+ fenced code blocks AND ("decision" OR "trade-off" OR "alternative" OR "we chose" OR "rationale")
       → type = technical-brief
    2. Source contains ("we propose" OR "we're proposing" OR "we recommend") AND has an audience-mention
       (e.g., "for our team", "for the CTO") AND has ask language ("greenlight", "approve",
       "let me", "fund", "pilot")
       → type = pitch
    3. Source contains slide-shaped fragments — numbered slides like "Slide 1", "Slide 2",
       OR "next slide" cues, OR a sequence of short H1 + bullet pairs
       → type = presentation
    4. Source is bullet/list-heavy (>50% of non-heading lines start with a list marker)
       AND contains ("agenda" OR "talking points" OR "meeting" OR "discuss")
       → type = meeting-prep
    5. Source has 4+ H2 headings AND ("how to" OR reference-shape language: "this section
       describes", "the system uses")
       → type = handbook
    6. None of the above → type = handbook (safe fallback — most general type)
    ```
