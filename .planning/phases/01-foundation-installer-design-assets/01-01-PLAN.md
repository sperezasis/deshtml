    ```bash
    #!/usr/bin/env bash
    # deshtml installer — atomic, idempotent, no-sudo curl-pipe-bash one-liner.
    # Source: https://github.com/sperezasis/deshtml
    set -euo pipefail

    REPO_URL="https://github.com/sperezasis/deshtml.git"
    RAW_VERSION_URL="https://raw.githubusercontent.com/sperezasis/deshtml/main/VERSION"
    DEST="$HOME/.claude/skills/deshtml"

    main() {
      # 1. Refuse root (D-05)
      if [ "${EUID:-$(id -u)}" -eq 0 ]; then
        echo "Do not run as root — deshtml installs into your home directory." >&2
        exit 1
      fi

      # 2. Fetch the pinned tag from the VERSION file on main (D-02)
      local version
      version="$(curl -fsSL "$RAW_VERSION_URL" | tr -d '[:space:]')"
      if [ -z "$version" ]; then
        echo "Could not read VERSION from $RAW_VERSION_URL" >&2
        exit 1
      fi

      # 3. Stage to a temp dir; trap guarantees cleanup if anything fails (D-04)
      local tmp
      tmp="$(mktemp -d 2>/dev/null || mktemp -d -t deshtml)"
      trap 'rm -rf "$tmp"' EXIT

      # 4. Shallow clone the pinned tag (D-02)
      echo "Installing deshtml v${version}..."
      git clone --depth 1 --branch "v${version}" "$REPO_URL" "$tmp/deshtml" >/dev/null 2>&1 || {
        echo "Failed to clone deshtml v${version} from $REPO_URL" >&2
        exit 1
      }

      if [ ! -d "$tmp/deshtml/skill" ]; then
        echo "Cloned repo is missing skill/ payload — refusing to install." >&2
        exit 1
      fi

      # 5. Atomic replace: stage payload next to DEST, swap, then delete old (D-03, D-04, D-06)
      mkdir -p "$(dirname "$DEST")"
      local stage="${DEST}.installing.$$"
      rm -rf "$stage"
      cp -R "$tmp/deshtml/skill" "$stage"

      if [ -d "$DEST" ]; then
        local backup="${DEST}.old.$$"
        mv "$DEST" "$backup"
        if mv "$stage" "$DEST"; then
          rm -rf "$backup"
        else
          # Roll back if the swap fails
          mv "$backup" "$DEST"
          echo "Atomic swap failed; existing install preserved." >&2
          exit 1
        fi
      else
        mv "$stage" "$DEST"
      fi

      # 6. Confirm
      echo "deshtml v${version} installed to $DEST"
      echo "Run /deshtml in Claude Code to start."
    }

    main "$@"
    ```

    **Acceptance-critical details:**
    - `set -euo pipefail` is the FIRST line after the shebang and comment block.
    - The script body is fully wrapped in `main() { ... }` and `main "$@"` is the last non-comment, non-blank line.
    - No `read` statements anywhere. No `/dev/tty` references.
    - No `mapfile`, `readarray`, `${var,,}`, `${var^^}`, or `declare -A`.
    - The `trap` line uses single quotes around the command string so `$tmp` is expanded at trap time, not at definition time. (Both work, but single quotes are the convention.)
    - The script never invokes `sudo`. Never `chown` outside the user's own files. Never writes outside `$HOME/.claude/skills/`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Public internet → user shell | The `curl -fsSL ... | bash` command pipes attacker-influenceable bytes (the install script) directly into a Bash interpreter running as the user. This is the entire blast radius of the install. |
| GitHub raw CDN → install.sh runtime | Once `install.sh` runs, it fetches the `VERSION` file and a `git clone` tag from `github.com`. Both are over HTTPS to GitHub. |
| User shell → user filesystem | `install.sh` writes only under `$HOME/.claude/skills/deshtml/` and to `mktemp -d` (which is under `$TMPDIR`, owned by the user). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | `bin/install.sh` served from raw.githubusercontent.com | mitigate | TLS only (curl `-fsSL` enforces fail-on-error; GitHub is HSTS-preloaded). README discloses what the script does in plain bullets so users can `curl ... -o install.sh && less install.sh && bash install.sh` if they prefer review-then-run. ASVS L1 V14.1 (build pipeline integrity) — out of V1 scope per CONTEXT.md (no checksum/signature in V1; documented in PITFALLS.md as accepted risk for curl-pipe-bash class). |
| T-01-02 | Denial of Service | Partial install on dropped network mid-clone | mitigate | All install logic is wrapped in `main() { ... }` invoked only on the LAST line of the script (D-04). If the script is truncated mid-download, `main` is never defined and Bash errors out before any filesystem writes. Additionally: writes go to `mktemp -d` first; the existing `~/.claude/skills/deshtml/` is only swapped at the end via `mv`, with rollback if swap fails. |
| T-01-03 | Elevation of Privilege | User runs `sudo bash install.sh` (or environment with `EUID=0`) | mitigate | Hard refusal at script entry (D-05): `if [ "${EUID:-$(id -u)}" -eq 0 ]; then echo "Do not run as root..."; exit 1; fi`. Acceptance criterion verifies the EUID check is present in `bin/install.sh`. |
| T-01-04 | Information Disclosure | Installer fetches additional payloads from third-party hosts at runtime | mitigate | Installer talks only to `github.com` and `raw.githubusercontent.com`. No third-party CDN, no analytics, no telemetry. Verified by absence of any URL in `bin/install.sh` outside the GitHub domains. |
| T-01-05 | Tampering | Floating-`main` install URL — a bad commit on `main` breaks every future install instantly | mitigate | Per D-02: `install.sh` reads `VERSION` from `main`, then `git clone --depth 1 --branch "v${VERSION}"` against the matching tag. The actual payload is pinned. Releases are advanced by bumping `VERSION` and pushing the matching tag — atomic from the user's view. |
| T-01-06 | Tampering | Shell-injection via crafted environment variables (e.g., `$HOME` contains `$(rm -rf ~)`) | mitigate | `set -euo pipefail` plus consistent double-quoting of every variable expansion in `bin/install.sh` (`"$HOME"`, `"$tmp"`, `"$DEST"`, `"v${version}"`). shellcheck CI (D-08) blocks regressions on any unquoted expansion. |
